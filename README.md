# Практический кейс: ETL-process  
## Описание: Загрузка CSV из СБИС, очистка, фильтрация по типам документов, базовый анализ, сохранение в Excel  
Запуск: Открыть в VS Code и запустить все ячейки  
Используемые библиотеки: Pandas, pathlib, glob, datetime  

Выгрузка из СБИС находится в папке Входящие. Загружаются все csv файлы в один датафрейм.  
Файлы с выгрузкой из аптек находятся в папке Аптеки/csv/correct/:  
Загружаются на обработку по одному файлу в датафрейм Pandas.  

В каждой строке делается проверка:  

1. Если "Поставщик" - ЕАПТЕКА, то к "Номер накладной" нужно добавляется /15.  
2. Ищутся все записи в выгрузке из СБИСа по данному номеру накладной  
3. Из найденных строк остаются только те, которые имеют один из типов документа: ["СчФктр", "УпдДоп", "УпдСчфДоп", "ЭДОНакл"]  
4. Если ничего не найдено - переход к следующей строке  
5. Если найдено - значения Номер, Сумма и Дата сохраняются  
6. Дата - в формате 01.07.2026  
7. В столбцы из пункта 5 записываются найденные для данной строки значения  
8. В столбец Сравнение дат ставится "Не совпадает!", если найденная дата и дата накладной отличаются. Иначе - пустая строка.  

Если по заданным условиям находится несколько подходящих документов - выбирается значения из первого найденного.  
Если дата счет фактуры отсутствует или пуста, а дата накладной заполнена, такая ситуация считается расхождением.  
Итоговый файл выгрузки по аптеке содержит такие столбцы в заданном порядке:  
Файл сохранен по пути Результат/{полная сегодняшняя дата}/{имя исходного файла без расширения} - результат.xlsx.
